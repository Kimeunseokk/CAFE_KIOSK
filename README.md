# ☕ Kiosk Ordering System (JAVA - PRECOURSE)

간단한 카페 키오스크 주문 프로그램입니다.  
사용자는 메뉴를 선택해 주문할 수 있으며, 관리자는 저장된 주문 내역을 조회·관리할 수 있습니다.

---

## 📌 프로그램 주요 기능

### 1️⃣ 일반 사용자 기능

#### ✔ 메뉴 출력  
프로그램 실행 시 전체 메뉴 목록을 보여줍니다.

#### ✔ 메뉴 주문  
- 메뉴 이름 입력  
- 수량 입력  
- 메뉴 존재 여부 검증  
- 수량 유효성 검증  
- 주문은 임시 장바구니(`OrderList`)에 저장됨

#### ✔ 장바구니 조회  
현재까지 주문한 모든 항목을 출력  
- 메뉴명  
- 수량  
- 해당 메뉴 총 금액  
- 전체 주문 금액 합산 출력

#### ✔ 주문 종료  
- 임시 `OrderList` 전체를 파일 기반 DB에 저장  
- 저장 후 장바구니 초기화  
- 서비스에서는 `System.exit()` 사용 ❌  
  - 컨트롤러에서 종료 제어

---

### 2️⃣ 관리자 모드 기능

#### ✔ 저장된 전체 주문 조회  
파일에서 모든 주문 내역 로드 후 출력  
- 주문번호  
- 주문된 메뉴 목록  
- 메뉴별 수량

#### ✔ 관리자 선택 메뉴  
- 1. 삭제 
- 2. 수정 (구현 예정)  
- 3. 관리자 모드 종료

---

## 📁 주요 클래스 구조

### **Controller**
- `KioskController`
  - 메뉴 입력, 기능 선택, 반복 제어
  - 사용자/관리자 기능 분기 처리

### **Service**
- `KioskService`
  - 주문 저장  
  - 장바구니 조회  
  - 파일 DB 저장  
  - 관리자 조회 기능 처리  
  - 실제 비즈니스 로직 담당

### **View**
- `InputView`: 사용자 입력 담당  
- `OutputView`: 출력 담당

### **Domain**
- `Menu`: Enum 메뉴 목록  
- `Order`: 단일 주문 객체  
- `OrderList`: 주문 리스트(장바구니)  
- `CafeOwner`(구 버전, 현재 비활성)

### **Repository**
- `FileOrderListRepository`
  - 주문 내역 파일 저장  
  - 주문 전체 불러오기  
  - List<OrderList> 단위로 관리

---

## 💾 주문 저장 방식

1. 사용자가 여러 개의 메뉴를 주문하면  
   → `OrderList` 객체에 하나의 주문 리스트로 저장

2. 주문 종료 시  
   → `OrderList` 하나를 파일에 저장  
   → 이후 장바구니 초기화

3. 관리자 모드에서  
   → 저장된 모든 `OrderList` 불러와서 출력
